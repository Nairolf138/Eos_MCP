# Dossier de preuves de re-licensing

Ce répertoire centralise les preuves de titularité et/ou d’accord explicite nécessaires pour sécuriser un re-licensing du projet.

## Convention de nommage

- `YYYY-MM-DD-contributor-audit.md` : audit des contributeurs (historique Git + fichiers majeurs).
- `YYYY-MM-DD-relicensing-decision.md` : décision opérationnelle (accord obtenu, ou remplacement/réécriture).
- `YYYY-MM-DD-third-party-<identifiant>.md` : preuve d’accord pour une contribution tierce identifiée.

## Politique de preuve minimale

Pour chaque contribution tierce identifiée, conserver :

1. l’identification du code concerné (fichiers, commits, lignes) ;
2. la preuve d’accord explicite de re-licensing (mail, ticket signé, clause contractuelle) ;
3. à défaut d’accord, la preuve de remplacement/réécriture (PR, commit, revues).
