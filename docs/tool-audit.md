## Méthode de classement du risque

- **faible (lecture)** : lecture, diagnostic, liste, ping ou contexte local.
- **moyen (action console)** : navigation/sélection ou action console qui ne programme pas directement un état de show.
- **élevé (déclenchement live)** : go, fire, bump, rappel, touche ou autre déclenchement immédiat.
- **élevé (modification show/live)** : patch, programmation, commande texte, niveau, DMX, record/update/label/set.

## Recoupement avec les documents existants

- `docs/tools.md` décrit les 118 outils exportés. Les sections utilisent le nom MCP entre parenthèses, ce qui permet le contrôle automatique.
- `docs/osc-coverage.md` contient également les 118 outils exportés. Les outils sans OSC direct sont documentés comme non OSC ou orchestrateurs.
- Le recoupement met surtout en évidence une différence de taxonomie : le fichier `src/tools/programming/index.ts` contient des outils de programmation dont la famille documentaire est parfois sémantique (`cues`, `palettes`, `patch`) plutôt que le répertoire source `programming`.

## Priorités recommandées

1. Ajouter les garde-fous (`dry_run`, `require_confirmation`, `commands_preview`) aux écritures live : DMX, submasters, effects et programming.
2. Aligner la taxonomie de `programming` avec les familles documentaires pour éviter les doublons `patch/cues/palettes` lors des audits.
3. Renforcer les parseurs de lecture (`macros`, `pixelMaps`, `patch`) avec fixtures de réponses EOS réelles et diagnostics de champs inconnus.
4. Maintenir le test de recoupement `docs/osc-coverage.md` pour éviter toute dérive entre export MCP et documentation.
