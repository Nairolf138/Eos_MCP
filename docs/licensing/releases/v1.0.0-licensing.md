# Release notes — `v1.0.0-licensing`

- **Version dédiée** : `v1.0.0-licensing`
- **Date de publication** : 2026-04-05 (UTC)
- **Date d’entrée en vigueur** : 2026-04-15 (UTC)

## Objectif de cette release

Cette release formalise le cadre de licensing dual (canal communautaire AGPLv3 + canal commercial propriétaire), et fixe une date d’effet explicite pour la politique commerciale et de conformité.

## Impact sur les utilisateurs existants

- Les utilisateurs déjà conformes à l’AGPLv3 peuvent continuer l’usage communautaire sans interruption.
- Les contrats commerciaux déjà signés restent applicables selon leurs clauses propres.
- Aucun changement technique obligatoire n’est imposé par cette release : le changement est principalement juridique et contractuel.
- Les utilisateurs existants qui exploitent EOS MCP dans un cadre fermé sans contrat commercial devront régulariser leur situation avant la date d’entrée en vigueur.

## Obligations pour les nouveaux usages commerciaux

À partir du 2026-04-15 (UTC), tout nouveau usage commercial fermé devra :

1. disposer d’un contrat de licence commerciale valide ;
2. respecter la grille tarifaire et les modalités de calcul applicables ;
3. conserver les notices de propriété intellectuelle exigées ;
4. respecter les obligations de reporting et d’audit prévues au contrat ;
5. aligner la distribution/monétisation avec le périmètre contractuel autorisé.

## Politique de compatibilité pour anciennes versions

- Les versions antérieures publiées restent consultables et traçables.
- La compatibilité technique ascendante/descendante n’est pas modifiée par cette release de gouvernance.
- Pour les versions antérieures utilisées dans un contexte commercial fermé, la conformité est appréciée selon les obligations de licence en vigueur à la date d’usage.
- En cas d’écart, la politique de mise en conformité décrite dans `docs/licensing/compliance.md` s’applique.
